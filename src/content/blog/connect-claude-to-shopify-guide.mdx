## Method 1: Add Your Store to Claude as a Connector (No Code)

Shopify stores expose a **Model Context Protocol (MCP) endpoint** — a standard way for AI assistants to query your storefront. Claude supports these as custom connectors.

1. **Find your MCP URL.** It follows the pattern `https://yourstore.com/api/mcp` (also works on your `.myshopify.com` domain).
2. **Open Claude's settings.** In Claude on web or desktop, go to **Settings → Connectors → Add custom connector**.
3. **Paste the MCP URL** and save. No API key is needed — the endpoint only exposes storefront data that is already public.
4. **Enable it in a chat.** Start a new conversation, open the tools/connectors menu, and toggle your store on.
5. **Test it.** Ask: *"Search our catalog for products under $50 and summarize what we sell."* Claude should return live product data from your store.

What this unlocks: catalog search, product detail lookups, price and availability checks, and cart assistance — useful for competitive research on your own assortment, drafting collection copy grounded in real products, and prototyping an AI shopping assistant experience.

What it does **not** unlock: orders, customers, inventory, or anything from the Shopify admin. For that, keep reading.

To make your storefront easier for Claude (and every other AI assistant) to understand, pair this with proper [agents.md and llms.txt files](/blog/shopify-agents-md-llms-txt-customize-ai-discovery) so AI tools can discover and interpret your store correctly.

## Method 2: Claude Desktop + an Admin MCP Server

To manage the store itself from a Claude conversation — update products, check orders, adjust inventory — you need an MCP server with **Admin API** access. This takes about 30 minutes and requires comfort with configuration files, but no real programming.

1. **Create a custom app in Shopify.** In your Shopify admin: **Settings → Apps and sales channels → Develop apps → Create an app**. Name it something like "Claude Admin Access."
2. **Grant only the scopes you need.** For a product-focused workflow: `read_products`, `write_products`. Add order or inventory scopes only if your workflow uses them — least privilege is the rule with AI-connected tokens.
3. **Install the app and copy the Admin API access token.** Shopify shows it once; store it securely.
4. **Add an MCP server to Claude Desktop.** In your `claude_desktop_config.json`, register a Shopify Admin MCP server (several open-source options exist on GitHub — search "shopify mcp server") with your store domain and token as environment variables.
5. **Restart Claude Desktop** and test: *"List our 10 most recently updated products and flag any missing descriptions."*

From here, real merchant workflows open up: bulk-rewriting product descriptions, spotting inventory anomalies, drafting weekly performance summaries. Our [Claude for Shopify small-business guide](/blog/claude-for-small-business-shopify-merchants) walks through the highest-ROI use cases if you want ideas.

**Security note:** an Admin token in an MCP server means Claude can act on your store within the granted scopes. Review changes before applying them in bulk, and rotate the token if you ever share your config.

## Method 3: Zapier or Make (Automation Without Code)

Methods 1 and 2 are conversational — you chat, Claude acts. If you want automations that run **without you present**, connect Claude and Shopify through an automation platform:

- **New product → Claude writes the description → product updated.** The classic three-step Zap.
- **Customer email arrives → Claude drafts a reply with your policies → draft lands in your inbox for review.**
- **Every Monday → Shopify order export → Claude analysis → summary email to your team.**

Zapier starts around $20/month; Make.com from $9/month and is usually cheaper at volume. Full walkthroughs of these workflows — including prompts and cost math — are in our [complete Claude + Shopify integration guide](/blog/shopify-claude-ai-integration-automation).

## Method 4: Direct API Integration (Developers)

For custom workflows at scale — hundreds of products, multi-step agents, app development — call the Claude API and Shopify Admin GraphQL API directly:

1. Get an Anthropic API key and pick a model (Sonnet for volume work, Opus-class for complex reasoning).
2. Use Shopify's **Admin GraphQL API** to read and write store data; the [Product Catalog API guide](/blog/shopify-product-catalog-api-guide) covers the product-data side in depth.
3. Batch work through Shopify's [bulk operations](/blog/shopify-bulk-operations-large-catalogs) to stay inside rate limits.

Budget guidance: generating 100 product descriptions runs a few dollars in API costs; a comprehensive integration (descriptions + support drafts + weekly analysis) typically lands at $15–40/month. ## Troubleshooting

- **Claude says it can't reach the connector.** Confirm the URL is exactly `https://yourstore.com/api/mcp` and your store is on a plan with the storefront live (password-protected dev stores block public MCP access).
- **Catalog results look stale.** MCP reads your live storefront; if products are missing, check they're published to your Online Store sales channel.
- **Admin MCP server won't start.** Nine times out of ten it's the token: regenerate it, confirm the scopes, and check the environment variable names match your server's README.
- **Automations produce generic copy.** Feed Claude your brand voice and constraints in the prompt template — the integration guide has [tested prompt patterns](/blog/shopify-claude-ai-integration-automation) for descriptions and support replies.
